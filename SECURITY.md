# Security policy

This is a public repository. Do not include credentials, personal data, production datasets, private operational details or internal architecture in issues, pull requests, commits or discussions.

## Reporting a security concern

Do not open a public issue for a suspected vulnerability or accidental data exposure.

Contact the repository owner through an existing approved private channel and include only the minimum information needed to identify the affected file or behaviour. Do not attach credentials, tokens, personal data or confidential datasets.

If sensitive information is accidentally committed:

1. Revoke or rotate any exposed credential immediately.
2. Contact the repository owner privately.
3. Do not copy the sensitive value into another commit, issue or pull request.

Public documentation should describe user-visible behaviour and maintenance procedures only.
